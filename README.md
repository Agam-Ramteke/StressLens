# StressLens

StressLens is a desktop web app for stress and sleep prediction. It trains two Random Forest models from the preserved wellness dataset, exposes predictions through a FastAPI backend, and provides a browser-based dashboard for entering daily signals and reading model explanations.

The dataset is kept at:

```text
backend/data/sleep_mobile_stress_dataset_15000.csv
```

## Features

- Predicts daily `stress_level` and `sleep_quality_score`.
- Explains each score with per-feature contribution bars.
- Validates API input ranges with Pydantic.
- Generates practical recommendations from model drivers and wellness thresholds.
- Runs as a local FastAPI backend plus a desktop web frontend.
- Keeps the Android Studio work shelved in `android-app/` for later.

## Quick Start

Train the models:

```bash
cd backend
pip install -r requirements.txt
python scripts/train_models.py
```

Start the backend:

```bash
uvicorn app.main:app --reload
```

Start the desktop web app in another terminal:

```bash
cd frontend
python -m http.server 5173
```

Open `http://localhost:5173`.

The old Expo prototype is still available if needed:

```bash
cd mobile
npm install
npm run web
```

## Backend API

Base URL during local development:

```text
http://localhost:8000
```

Endpoints:

- `GET /health` - backend, dataset, and model status.
- `GET /meta` - supported fields, categories, feature names, and training metrics.
- `POST /predict` - stress and sleep prediction with explanations.

Example prediction request:

```json
{
  "age": 28,
  "gender": "Male",
  "occupation": "Software Engineer",
  "daily_screen_time_hours": 7,
  "phone_usage_before_sleep_minutes": 60,
  "sleep_duration_hours": 6.5,
  "caffeine_intake_cups": 2,
  "physical_activity_minutes": 30,
  "notifications_received_per_day": 150,
  "mental_fatigue_score": 6
}
```

## Model Training

The training script reads the CSV from `backend/data/`, label-encodes `gender` and `occupation`, trains separate Random Forest regressors for stress and sleep, then writes artifacts to `backend/artifacts/`.

Generated files:

- `stress_model.joblib`
- `sleep_model.joblib`
- `encoders.joblib`
- `metadata.json`

`backend/artifacts/` is ignored by git because these files are generated locally.

## Desktop Web App

The current frontend lives in `frontend/`.

It is set up for:

- Desktop-first browser use.
- No build step and no npm dependencies.
- Backend URL editing in the interface.
- Local form persistence with `localStorage`.
- Stress and sleep score cards.
- Feature contribution bars for model explanations.
- Recommendation rendering from the backend response.

Run it with:

```bash
cd frontend
python -m http.server 5173
```

Then open:

```text
http://localhost:5173
```

## Notes

- Train models before starting the API, or `/health` will report `model_loaded: false`.
- The API is permissive about CORS for local development.
- Predictions are wellness estimates from the dataset and are not medical advice.
