# StressLens

AI-powered stress and sleep analysis dashboard. Enter your daily habits and get real-time predictions, per-feature explanations, and actionable wellness recommendations — all from your browser.

> **Note:** This web app is a working demo. The long-term goal is a native Android app (shelved in `android-app/`) that pulls data from Health Connect, Usage Stats, and Notification Listener automatically. The current dashboard lets you validate the models and UX before building that out.

## Live Demo

Deployed on Render (free tier — first request may take ~30 s to cold-start):

```
https://stresslens.onrender.com
```

## Features

- **Dual predictions** — stress level and sleep quality score (1–10) from two Random Forest regressors.
- **Per-feature explanations** — each factor ranked by how much it pushed the score up or down, with health-aware copy instead of raw SHAP values.
- **Mental fatigue quiz** — a five-question in-app quiz that computes your fatigue score automatically.
- **Wellness Index** — a combined 0–100 score with comparison bars against dataset averages.
- **Lifestyle radar chart** — visual balance across sleep, activity, screen time, caffeine, mental load, and digital hygiene.
- **Actionable recommendations** — threshold-based tips generated from model drivers and your inputs.
- **Profile & history** — saved in `localStorage` with ZIP export.
- **Dark / light theme** — toggle in the settings drawer.
- **Single-container deployment** — one Docker image serves the API and static frontend.

## Quick Start (Local)

### 1. Train the models

```bash
cd backend
pip install -r requirements.txt
python scripts/train_models.py
```

This reads the CSV, label-encodes `gender` and `occupation`, trains separate Random Forest regressors for stress and sleep, and writes four artifacts to `backend/artifacts/`:

| File | Purpose |
|---|---|
| `stress_model.joblib` | Stress regressor |
| `sleep_model.joblib` | Sleep regressor |
| `encoders.joblib` | Label encoders for categorical features |
| `metadata.json` | Feature means, class lists, training metrics |

### 2. Start the backend

```bash
cd backend
uvicorn app.main:app --reload
```

The API runs at `http://localhost:8000`.

### 3. Start the frontend

```bash
cd frontend
python -m http.server 5173
```

Open `http://localhost:5173`. The frontend auto-detects `localhost:5173` and points API calls to `http://localhost:8000`.

### Running via Docker

```bash
docker build -t stresslens .
docker run -p 10000:10000 stresslens
```

Open `http://localhost:10000`. The frontend and API are served from the same origin — no extra configuration needed.

## API Reference

| Method | Path | Description |
|---|---|---|
| `GET` | `/health` | Backend, dataset, and model status |
| `GET` | `/meta` | Supported fields, categories, feature names, training metrics |
| `POST` | `/predict` | Stress and sleep prediction with explanations |

### Example request — `POST /predict`

```json
{
  "age": 28,
  "gender": "Male",
  "occupation": "Software Engineer",
  "daily_screen_time_hours": 7,
  "phone_usage_before_sleep_minutes": 60,
  "sleep_duration_hours": 6.5,
  "caffeine_intake_cups": 2,
  "physical_activity_minutes": 30,
  "notifications_received_per_day": 150,
  "mental_fatigue_score": 6
}
```

### Response highlights

- `stress_level` / `sleep_quality_score` — predicted scores (1–10).
- `stress_band` / `sleep_band` — human-readable band (Low / Moderate / High, Poor / Fair / Good).
- `stress_explanations` / `sleep_explanations` — per-feature contribution, sorted by impact.
- `recommendations` — up to four actionable tips.
- `base_stress_score` / `base_sleep_score` — dataset-average baselines for comparison.

## Deployment (Render)

The repo includes a `Dockerfile` and `render.yaml` for one-click deployment on [Render](https://render.com):

1. Connect the GitHub repo to Render.
2. Render detects `render.yaml` and creates a **Docker web service** on the free tier.
3. The container serves both the API and the static frontend from a single Uvicorn process on port `10000`.

### Environment variables

| Variable | Default | Description |
|---|---|---|
| `PORT` | `10000` | Listening port (set by Render) |
| `ALLOWED_ORIGINS` | `*` | Comma-separated CORS origins |
| `DEBUG` | _(unset)_ | Set to any value to enable `/docs` and `/redoc` |

## Notes

- Train models before starting the API, or `/health` will report `model_loaded: false`.
- The API is permissive about CORS for local development — tighten `ALLOWED_ORIGINS` in production.
- Predictions are wellness estimates from the dataset and are **not medical advice**.
